## 🔍 문제 분석

### 발견된 문제
- **CSS Variant 스타일이 적용되지 않음**: HTML에서 `variant-unified` 클래스를 사용하고 있으나, 해당 variant의 스타일이 CSS에 없음
- **기본 스타일만 존재**: 공통 스타일만 있고 variant별 특화 스타일 부재
- **디자인 시스템 변수 매핑 누락**: 제안된 CSS 변수(`--canvas-*`, `--accent-*`)가 정의되지 않음

### 근본 원인
1. **확장성 문제**: Variant 시스템이 설계되었으나 구현되지 않음
2. **일관성 문제**: 디자인 토큰 매핑이 누락됨
3. **협업 코드 문제**: 디자이너가 제안한 variant가 개발에 반영되지 않음

## ✅ 적용된 개선 사항

### 1. CSS 변수 매핑 (일관성)

```css
:root {
  /* 디자인 variant용 변수 (기존 변수에 매핑) */
  --canvas-bg: var(--bg);
  --canvas-surf: var(--surface);
  --canvas-text: var(--text-1);
  --canvas-dim: var(--text-3);
  --canvas-border: var(--border);
  --accent: var(--pwc-orange);
  --accent-dim: var(--accent-alpha-12);
  --panel-bg: var(--surface);
  --radius: var(--r);
}
```

**CTO 관점**:
- ✅ 기존 디자인 토큰과 호환성 유지
- ✅ 새로운 변수는 기존 변수에 매핑하여 일관성 보장
- ✅ 향후 디자인 시스템 변경 시 한 곳만 수정하면 됨

### 2. Variant 시스템 구현 (확장성)

#### Variant 1: 미니멀 스피너 중심
- **용도**: 간단한 로딩 화면
- **특징**: 스피너만 표시, 프로그레스바/단계 인디케이터 숨김
- **확장성**: 빠른 로딩 시 사용

#### Variant 2: 프로그레스바 중심
- **용도**: 진행률이 중요한 경우
- **특징**: 프로그레스바 + 퍼센트 표시
- **확장성**: 파일 업로드 등 진행률 표시 필요 시 사용

#### Variant 3: 단계별 진행 디자인
- **용도**: 단계별 진행이 중요한 경우
- **특징**: 단계 인디케이터만 표시
- **확장성**: 멀티스텝 프로세스에 적합

#### Variant 4: 통합 인디케이터 (하이브리드) ⭐ 현재 사용 중
- **용도**: 현재 그래프 로딩 화면
- **특징**: 스피너 + 단계 인디케이터 결합, 프로그레스바 제거
- **UX 개선**:
  - 단계 인디케이터 배경 강조 (`rgba(226,219,208,0.35)`)
  - 활성 단계 시각적 강조 (scale, 배경색, 펄스 애니메이션)
  - 완료된 단계 배경 강조
  - 연결선 그라데이션 효과
  - 텍스트 그림자로 가독성 향상

#### Variant 5: 스켈레톤 UI (고급)
- **용도**: 향후 고급 로딩 UI
- **특징**: 실제 콘텐츠 구조를 미리 보여줌
- **확장성**: 향후 구현 예정

### 3. 애니메이션 개선 (UX)

#### 부드러운 전환
```css
/* 기존 */
animation: loading-spin .8s linear infinite;

/* 개선 */
animation: spin 1s cubic-bezier(0.4, 0, 0.2, 1) infinite;
transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);
```

**CTO 관점**:
- ✅ 자연스러운 애니메이션 (cubic-bezier 사용)
- ✅ 일관된 easing 함수 사용
- ✅ 성능 최적화 (GPU 가속 활용)

#### 펄스 애니메이션
```css
@keyframes pulse-dot {
  0%, 100% {
    box-shadow: 0 0 0 3px var(--accent-alpha-12),
                0 0 12px var(--accent-alpha-12);
  }
  50% {
    box-shadow: 0 0 0 6px var(--accent-alpha-12),
                0 0 20px var(--accent-alpha-12);
  }
}
```

**CTO 관점**:
- ✅ 활성 단계 시각적 강조
- ✅ 사용자 주의 집중
- ✅ 성능 고려 (box-shadow 애니메이션)

### 4. 시각적 계층 구조 개선 (디자이너 관점)

#### 단계 인디케이터 개선
- **배경 강조**: 완료/활성 단계에 배경색 추가
- **크기 변화**: 활성 단계 dot 크기 증가 (8px → 10px)
- **그림자 효과**: 활성 단계 펄스 애니메이션
- **연결선 그라데이션**: 완료된 단계의 연결선에 그라데이션 적용

#### 텍스트 가독성 향상
- **텍스트 그림자**: `text-shadow: 0 1px 1px rgba(0,0,0,0.06)`
- **자간 조정**: `letter-spacing: -0.02em`
- **계층 구조**: 메인 텍스트와 가이던스 텍스트 명확한 구분

### 5. 반응형 디자인 (호환성)

```css
@media (max-width: 768px) {
  .loading-spinner { width: 40px; height: 40px; }
  .loading-text { font-size: 13px; }
  .loading-guidance { font-size: 11px; }
  .loading-progress { width: 200px; }
  .loading-overlay.variant-unified .loading-steps {
    padding: 10px 16px;
    gap: 8px;
  }
}
```

**CTO 관점**:
- ✅ 모바일 환경 지원
- ✅ 터치 인터페이스 고려
- ✅ 작은 화면에서도 가독성 유지
