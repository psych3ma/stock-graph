# 첫 화면 UX 문제 CTO 검토

## 발견된 문제

### 1. 노드 겹침 (Critical)
- **현상**: 수십 개 노드가 한 곳에 겹쳐서 라벨이 읽을 수 없음
- **원인**: `initPositions()`가 단순 원형 배치만 수행, 노드 간 거리 계산 없음
- **영향**: 사용자가 그래프를 탐색할 수 없음 → 서비스 사용 불가

### 2. 레이아웃 알고리즘 부재 (Critical)
- **현상**: Force-directed layout이나 스프링 레이아웃 없음
- **원인**: 정적 배치만 구현됨
- **영향**: 대용량 그래프에서 확장 불가

### 3. 초기 뷰 설정 미흡 (High)
- **현상**: 로드 후 자동 fit-to-view 없음, 줌 레벨이 부적절
- **원인**: `loadGraph()` 후 `resetView()` 호출 안 함
- **영향**: 사용자가 수동으로 줌/패닝해야 함

### 4. 사이드바 초기 상태 (Medium)
- **현상**: 패널이 닫혀있거나 보이지 않음
- **원인**: 초기 상태 불명확
- **영향**: 노드 상세/채팅 기능 접근 불가

## 해결 방안

### 즉시 적용 (Quick Fix)
1. **격자 기반 초기 배치**: 원형 대신 격자로 배치해 겹침 최소화
2. **자동 fit-to-view**: 그래프 로드 후 전체 노드가 보이도록 줌/패닝 자동 조정
3. **사이드바 초기 상태 명확화**: 항상 열린 상태로 시작

### 중기 개선 (Next Sprint)
1. **Force-directed layout**: D3.js forceSimulation 또는 간단한 물리 시뮬레이션
2. **클러스터링**: 노드 타입별 그룹화
3. **LOD (Level of Detail)**: 줌 레벨에 따라 노드 수 조절

### 장기 개선 (Future)
1. **WebGL 렌더링**: 대용량 그래프 (1000+ 노드)
2. **서버 사이드 레이아웃**: Neo4j GDS (Graph Data Science) 활용
