# 노드 겹침 문제 CTO 검토

## 현재 상태 분석

### 문제점
1. **50개 노드가 한 화면에 겹쳐서 읽을 수 없음**
   - 초기 로드가 너무 많음 (50 nodes + 100 edges)
   - Force-directed 알고리즘이 약함 (20회 반복, 힘 상수 낮음)
   - 격자 배치가 좁게 설정됨

2. **텍스트 라벨 겹침**
   - 노드 라벨이 서로 겹쳐서 읽을 수 없음
   - 긴 회사명이 잘림 (`...`)

3. **초기 뷰 최적화 부족**
   - fitToView가 제대로 작동하지 않음
   - 줌 레벨이 부적절함

## 해결 전략

### 즉시 적용 (Critical)
1. **초기 로드 수 감소**: 50 → 20 nodes, 100 → 30 edges
2. **Force-directed 강화**: 반복 20 → 50회, 힘 상수 증가
3. **격자 간격 증가**: 최소 거리 60px → 100px
4. **노드 라벨 개선**: 겹치지 않도록 위치 조정 또는 숨김

### 중기 개선
1. **점진적 로딩**: 초기 20개만, 스크롤/확대 시 추가 로드
2. **클러스터링**: 노드 타입별 그룹화 강화
3. **라벨 최적화**: 줌 레벨에 따라 라벨 표시/숨김
