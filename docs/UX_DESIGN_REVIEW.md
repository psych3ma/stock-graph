#### ✅ 개선 완료
- **변경 전**: "회사", "개인주주", "최대주주", "기관" (영문 타입명 없음, 개수 없음)

**레퍼런스 이미지 분석**:
- 노드 타입별로 색상 아이콘 + 영문 타입명 + 개수 표시
- 예: Company (12,304), Person (48,720), MajorShareholder (9,182), Stockholder (61,004)

**현재 구현**:

**UX 권장사항**:

- 개수 표시로 사용자가 데이터 규모를 즉시 파악 가능
- 색상 아이콘으로 시각적 구분 명확

### 2. 관계 (Relationship) 표시

#### 레퍼런스 분석
- 관계 타입별로 화살표 아이콘 + 영문 타입명 + 개수 표시
- 예: HOLDS_SHARES (83,441), HAS_COMPENSATION (12,304)

#### 현재 구현 상태
- 관계는 엣지로만 표시됨
- 관계 타입별 개수 표시 없음
- 관계 범례(Legend) 없음

#### 🎯 UX 개선 제안

**우선순위: Medium**

1. **관계 범례 추가**
   - 사이드바 또는 상단 바에 관계 타입별 개수 표시
   - HOLDS_SHARES: 화살표 아이콘 + 개수
   - 필요 시 HAS_COMPENSATION 등 추가 관계 타입 표시

2. **시각적 일관성**
   - 노드 필터와 동일한 스타일 적용
   - 화살표 아이콘 사용 (레퍼런스와 동일)

**구현 예시**:
```html
<div class="relationship-legend">
  <span style="font-size:11px;color:var(--text-3);">관계:</span>
  <div class="rel-item">
    <svg>...</svg> <!-- 화살표 아이콘 -->
    <span>HOLDS_SHARES</span>
    <span class="rel-count">83,441</span>
  </div>
</div>
```

### 3. 답변 신뢰도 (Answer Reliability)

#### 레퍼런스 분석
- **DB**: 주황색 태그 + "Neo4j 직접 조회"
- **EMPTY**: 노란색 태그 + "쿼리 실행, 결과 없음"
- **LLM**: 연한 빨간색 태그 + "추론 (환각 주의)"

#### 현재 구현 상태
- ✅ 신뢰도 태그 표시 (DB, DB_EMPTY, LLM)
- ⚠️ 색상 및 설명이 레퍼런스와 다름
- ⚠️ "환각 주의" 경고가 명확하지 않음

#### 🎯 UX 개선 제안

**우선순위: High**

1. **색상 체계 정렬**
   ```css
   /* 현재 */
   .src-db { background: #...; }
   .src-db-empty { background: #...; }
   .src-llm { background: #...; }
   
   /* 권장 (레퍼런스 기반) */
   .src-db { 
     background: #fff4ed; /* 주황 계열 */
     border-color: #fbc99a;
     color: #d85604;
   }
   .src-db-empty { 
     background: #fffbf0; /* 노란색 계열 */
     border-color: #f5d896;
     color: #b8860b;
   }
   .src-llm { 
     background: #fff0f0; /* 연한 빨간색 */
     border-color: #f5b7b7;
     color: #d85604;
   }
   ```

2. **설명 텍스트 개선**
   - **DB**: "Neo4j 직접 조회" (현재: "DB")
   - **DB_EMPTY**: "쿼리 실행, 결과 없음" (현재: "DB (없음)")
   - **LLM**: "추론 (환각 주의)" (현재: "LLM")

3. **시각적 강조**
   - LLM 태그에 경고 아이콘 추가 (⚠️)
   - LLM 태그에 더 강한 시각적 경고 (배경색, 테두리)

**구현 예시**:
```html
<span class="src-tag src-llm">
  ⚠️ 추론 (환각 주의)
</span>
```

## 🎨 전체적인 UX 개선 권장사항

### 1. 정보 계층 구조
- **1차**: 노드/관계 타입 (색상 아이콘)
- **2차**: 라벨 (한글명)
- **3차**: 개수 (회색, 작은 폰트)

### 2. 색상 일관성
- 노드 타입 색상과 필터 아이콘 색상 일치
- 답변 신뢰도 색상 체계 명확화

### 3. 가독성
- 개수는 천 단위 구분 (12,304)
- 작은 폰트(10px)로 정보 밀도 조절
- 회색 계열로 2차 정보 표시

### 4. 접근성
- 색상만으로 구분하지 않고 텍스트 라벨 제공
- 충분한 색상 대비 (WCAG AA 기준)
