# UX: 데이터 로딩 중 빈 화면 개선

## 문제

- "데이터 로딩 중..." 문구만 상단에 있고, **그래프 영역은 완전히 빈 칸**이라 사용자가 무엇이 진행 중인지 알기 어려움.
- 노드 유형이 "0 건"으로만 보여 불안감·어리둥절함 유발.

## 개선 사항 (UX 디자이너 관점)

### 1. 그래프 영역 로딩 오버레이

- **위치**: 그래프 캔버스 위에 반투명 오버레이를 두어, "지금 이 영역에서 무언가 진행 중"임을 명확히 표시.
- **구성**:
  - **스피너**: 회전 애니메이션으로 진행 중임을 시각적으로 전달.
  - **단계 문구**: 현재 무엇을 하고 있는지 한 줄로 표시.
  - **보조 문구**: "잠시만 기다려 주세요" 등 짧은 안내.

### 2. 단계별 메시지 (기대감·신뢰 제공)

| 단계 | 메인 문구 | 보조 문구 |
|------|-----------|-----------|
| 1 | 연결 확인 중... | Backend 서버에 연결합니다 |
| 2 | 데이터 로딩 중... | 노드·관계 데이터를 가져옵니다 |
| 3 | 노드 로딩 중... | 연결된 노드 정보를 불러옵니다 |
| 4 | 레이아웃 계산 중... | 노드 위치를 계산합니다 (잠시 걸릴 수 있습니다) |
| 5 | 렌더링 중... | 그래프를 그리는 중입니다 |

→ 로딩이 길어져도 **어느 단계에서 시간이 걸리는지** 알 수 있음.

### 3. 적용 내용

- **CSS**: `.graph-loading-overlay`, `.graph-loading-spinner`, `@keyframes graph-load-spin` 추가.
- **HTML**: `#graphLoadingOverlay` (스피너 + 단계 문구 + 보조 문구)를 그래프 영역 안에 추가.
- **JS**: `showGraphLoading(stepText, hintText)`, `hideGraphLoading()` 추가. `loadGraph()` 각 단계에서 `showGraphLoading` 호출, 완료·에러 시 `hideGraphLoading()` 호출.

### 4. 효과

- 빈 화면 대신 **항상 진행 중인 상태**가 보여 혼란 감소.
- 단계별 문구로 **무엇이 진행 중인지** 인지 가능.
- 스피너로 **대기 시간 예상**에 도움.
